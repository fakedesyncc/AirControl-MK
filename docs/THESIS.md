# AirControl — методические материалы для магистерской работы

Документ связывает реализованные возможности с типичной структурой магистерской
диссертации и описывает методику экспериментов. Тема:

> **Мультимодальная система бесконтактного управления ПК на основе слияния
> жестов, движения и голоса.**

---

## 1. Актуальность и новизна

**Проблема.** Традиционные манипуляторы (мышь, тачпад) непригодны в ряде сценариев:
ассистивные технологии (ограниченная моторика), стерильные среды (медицина),
интерактивные стенды, AR/VR. Существующие жестовые системы либо требуют
специального оборудования (Leap Motion, Kinect), либо неточны.

**Новизна работы:**
1. Слияние трёх модальностей (жест + движение + голос) в едином координаторе
   с разрешением конфликтов — вместо использования модальностей по отдельности.
2. Применение адаптивного **One-Euro фильтра** к жестовому позиционированию
   курсора и его количественное сравнение с EMA и фильтром Калмана.
3. Сменный распознаватель жестов (эвристика ↔ обучаемая модель) и сравнение
   подходов на едином стенде.
4. Иерархия жестов: статические позы, **динамические траектории** (свайпы) и
   **бимануальные** жесты (двуручный pinch-to-zoom) в одной системе.
5. Стабилизация распознавания позы взвешенным по уверенности голосованием по
   окну кадров + доверительный гейт для ML-модели.
6. Только серийная веб-камера — без специализированного оборудования.
7. Ассистивные пресеты под разные ограничения моторики: базовый режим,
   тремор/дрожание и малый диапазон движения руки.
8. Опциональный офлайн-голос через Vosk без скрытого fallback в онлайн-сервис.

## 3. Методика экспериментов

### 3.1. Эксперимент 1 — стабилизация курсора (без участников)

Цель: выбрать фильтр с лучшим компромиссом «дрожание/задержка».

```bash
python -m aircontrol bench
```

Метрики: jitter (СКО координаты в покое) и latency (время выхода на 90% ступени).
Синтетический сигнал воспроизводим (фиксированный seed). Пример результата:

| Фильтр | jitter (СКО) | latency (мс) |
|--------|--------------|--------------|
| none | 0.01049 | 0 |
| ema | 0.00265 | 233 |
| **one_euro** | **0.00186** | **167** |
| kalman | 0.00232 | 250 |

Вывод: One-Euro доминирует EMA по обоим показателям → используется по умолчанию.

### 3.2. Эксперимент 2 — закон Фиттса, жесты vs мышь (с участниками)

Цель: количественно сравнить жестовое управление с обычной мышью.

```bash
python -m aircontrol fitts --method mouse   --participant P01   # контроль
python -m aircontrol fitts --method gesture --participant P01   # жесты
```

Стенд по ISO 9241-9 (multidirectional tapping): мишени по кругу, варьируются
амплитуда A и ширина W. Для каждого условия считается:
- ID = log₂(A/W + 1) — индекс сложности;
- MT — время наведения;
- TP = IDe/MT — **пропускная способность (бит/с)** — главный показатель;
- доля промахов (error rate).

Результаты пишутся в `aircontrol/data/logs/fitts_*.csv` (колонки: participant,
method, condition, ID, MT, throughput, error_rate, …). Рекомендуется ≥10 участников,
within-subject дизайн, рандомизация порядка методов, регрессия MT = a + b·ID
(проверка соответствия закону Фиттса, R²).

### 3.3. Эксперимент 3 — эвристика vs ML распознавание жестов

```bash
python -m aircontrol collect                 # сбор размеченного датасета
python -m aircontrol train --backend rf      # обучение + accuracy + confusion matrix
python -m aircontrol train --backend mlp
python -m aircontrol train --backend knn
```

Метрики: accuracy на отложенной выборке, confusion matrix, обобщение на новых
пользователей (leave-one-subject-out при сборе данных от нескольких людей).
Признаки инвариантны к сдвигу/масштабу/повороту (см. `features.extract_features`).

### 3.4. Графики для работы

Все рисунки генерируются одной командой (matplotlib, headless):

```bash
python -m aircontrol report
```

Создаются PNG в `aircontrol/data/figures/`: матрица ошибок ML, PCA пространства
признаков (разделимость поз), сравнение фильтров (jitter/latency), регрессия
закона Фиттса `MT = a + b·ID` с R² и пропускной способностью по методам, график
FPS из телеметрии. Если данных нет — модуль генерирует синтетический датасет для
демонстрации (это помечается в выводе).

### 3.5. Эксперимент 4 — производительность

Телеметрия (`evaluation/metrics.py`) пишет FPS, задержку кадра и задержку
детекции в CSV (`data/logs/telemetry_*.csv`). Сравнение нагрузки CPU при разных
фильтрах/распознавателях и на разном оборудовании.

### 3.6. Эксперимент 5 — ассистивная пригодность

Цель: проверить, что система полезна не только технически, но и как средство
доступа к компьютеру.

Условия:
- `balanced` — базовый ассистивный пресет;
- `steady` — пресет для тремора: сильнее сглаживание, dwell `steady`, cooldown;
- `low_motion` — пресет для малого диапазона движения кисти.

Задачи для участника:
- навести курсор и выполнить dwell-click по крупной цели;
- открыть текстовое поле и набрать короткую фразу экранной клавиатурой ОС;
- прокрутить страницу или список;
- сохранить ZIP-отчёт диагностики с помощью помощника или самостоятельно.

Метрики:
- время выполнения задачи;
- число ошибочных кликов;
- число повторных dwell-кликов после одного намерения;
- средний FPS и задержка детекции;
- субъективная оценка NASA-TLX или короткая SUS-анкета;
- качественная обратная связь по понятности стартового окна.

Этическое ограничение: тестирование с людьми с ОВЗ проводится только с
добровольным согласием, возможностью остановить тест в любой момент и без
сбора медицинских данных в репозиторий.

## 4. Возможные направления развития (раздел «Перспективы»)

- Добавление модальности **взгляда** (eye-tracking) в координатор слияния.
- Улучшение офлайн-распознавания речи: мастер установки модели Vosk и выбор языка.
- Динамические жесты (свайпы) через временные модели (LSTM/TCN).
- Пользовательское исследование удобства (SUS, NASA-TLX) в дополнение к Фиттсу.
- Экранная клавиатура и сканирующее меню действий для пользователей, которым
  недоступны щипки и быстрые движения.
