# AirControl

AirControl - приложение для управления компьютером с веб-камеры: курсор,
клики, прокрутка, часть клавиш и простые голосовые команды. Главный сценарий -
помочь человеку пользоваться ПК без обычной мыши или с минимальным движением
рук.

Проект ориентирован на обычного пользователя. Для запуска готовой сборки не
нужны Python, компиляторы или работа с консолью.

## Возможности

- управление курсором движением руки перед камерой;
- клики жестом щипка или dwell-click по удержанию курсора;
- безопасный режим тренировки без реальных кликов и клавиш;
- ассистивный профиль для слабых ПК и пользователей с ограниченной моторикой;
- калибровка активной зоны руки и порогов щипка;
- диагностика камеры, модели, прав ОС и input backend без терминала;
- ZIP-отчёт поддержки для разбора проблем у тестера;
- поддержка Windows, macOS и Linux через PyInstaller-сборки;
- Linux fallback для ввода через `pynput`, `xdotool` и `ydotool`.

## Установка для пользователя

Скачайте готовый артефакт из GitHub Actions или Releases:

- Windows: `AirControl-Setup.exe`;
- macOS: `AirControl-macOS.zip`;
- Debian/Ubuntu: `AirControl-Linux-amd64.deb`;
- Linux portable: `AirControl-Linux-x86_64.AppImage`.

После запуска откроется стартовое окно с крупными кнопками:

1. Безопасная тренировка.
2. Ассистивное управление.
3. Калибровка под пользователя.
4. Проверка системы.
5. ZIP-отчёт диагностики.
6. Просмотр камеры.

Рекомендуемый первый запуск - безопасная тренировка или просмотр камеры. В этих
режимах приложение показывает руку и жесты, но не отправляет клики и клавиши в
операционную систему.

## Важные разрешения

Windows:
- при первом запуске может появиться SmartScreen, потому что сборка не подписана;
- для обычной работы достаточно камеры и стандартных прав пользователя.

macOS:
- разрешите доступ к Camera, Microphone и Accessibility;
- если приложение не подписано, первый запуск: правый клик по `AirControl.app`,
  затем `Open`.

Linux:
- для камеры пользователь должен иметь доступ к видеоустройству;
- на Debian/Ubuntu может понадобиться `libgl1`, `libglib2.0-0`, `xdotool`;
- Wayland может блокировать глобальное управление мышью и клавиатурой;
- для Wayland можно использовать `ydotool`, если настроен `ydotoold` и доступ к
  `/dev/uinput`;
- если управление не работает, попробуйте Xorg-сессию и запустите проверку
  ввода мыши из окна диагностики.

## Статусы в окне

- `VIEW` - безопасный просмотр, реальные клики отключены.
- `CONTROL` - управление включено.
- `SAFE` - ввод отключён вручную, камера и жесты продолжают работать.
- `INPUT RISK` - текущая ОС или Linux-сессия может блокировать ввод.
- `INPUT ERROR` - жест распознан, но backend ввода отклонил действие.
- `Low FPS` или `Slow detection` - приложение снижает нагрузку на слабом ПК.

Если камера работает, но курсор не двигается, проблема обычно не в жесте, а в
правах ОС или input backend. Нажмите `Проверить систему` и сохраните ZIP-отчёт.

## Запуск из исходников

Исходники нужны разработчику. Пользователю лучше брать готовую сборку.

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python -m aircontrol
```

Windows:

```powershell
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python -m aircontrol
```

Полезные команды:

```bash
python -m aircontrol doctor --no-camera
python -m aircontrol support --no-camera
python -m aircontrol assistive --dry-input
python -m aircontrol assistive
python -m unittest discover -s tests
```

Optional-зависимости для исследований, расширенного ML и офлайн-голоса вынесены
в `requirements-optional.txt`.

## Горячие клавиши

Работают в английской и русской раскладке. Если фокус потерян, кликните по окну
камеры.

| Клавиша | Действие |
| --- | --- |
| `1` / `2` | просмотр / управление |
| `+` / `-` | чувствительность |
| `f` / `а` | фильтр курсора |
| `g` / `п` | эвристика / ML |
| `d` / `в` | dwell-click |
| `l` / `д` | скелет руки |
| `h` / `р` | HUD |
| `F2` / `F3` | тест Фиттса |
| `Esc` | выход |

## Жесты

- большой + указательный: левый клик или перетаскивание;
- большой + средний: правый клик;
- большой + безымянный: Backspace;
- большой + мизинец: Enter;
- два быстрых щипка: альтернативное действие для выбранного пальца;
- открытая ладонь: пауза курсора;
- жест `мир` с движением: прокрутка;
- кулак: голосовая команда;
- две руки щипком: zoom, если включён двуручный режим.

Для пользователя, которому сложно делать щипок, включайте ассистивный режим с
dwell-click: клик выполняется удержанием курсора на месте.

## Сборка релиза

Подробно: [BUILD.md](BUILD.md).

Локальная проверка исходников:

```bash
python -m compileall aircontrol tests tools packaging/pyinstaller_hooks run_app.py
python -m unittest discover -s tests
python -m aircontrol doctor --no-camera
```

Сборка под текущую ОС:

```bash
pip install -r requirements-build.txt
pyinstaller aircontrol.spec --noconfirm
python tools/smoke_build.py
```

PyInstaller не кросс-компилирует. Для Windows, macOS и Linux используйте
workflow `.github/workflows/build.yml`.

## Структура

```text
aircontrol/
  app.py              главный runtime
  launcher.py         стартовое окно
  config.py           конфигурация и профили
  tracking/           камера, MediaPipe, фильтры
  gestures/           эвристика, ML, динамические жесты
  control/            курсор, клики, backend ввода
  platform/           Windows, macOS, Linux API
  diagnostics.py      doctor и support bundle
  ui/                 HUD и калибровка
tests/                unit-тесты
packaging/            PyInstaller, Linux, Windows packaging
tools/                smoke и release verification
```

## Ограничения

- Неподписанные сборки могут требовать ручного подтверждения запуска.
- На Linux Wayland не всегда разрешает глобальный ввод. Это ограничение
  окружения, а не распознавания жестов.
- Голосовые команды зависят от микрофона и выбранного backend.
- Качество управления зависит от света, положения камеры и стабильности FPS.

## Пользовательские данные

Локальные настройки, логи, датасеты жестов, скриншоты и ZIP-отчёты не
коммитятся. В собранном приложении они хранятся в пользовательском каталоге
AirControl.
