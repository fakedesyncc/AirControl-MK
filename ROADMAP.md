# AirControl Roadmap

AirControl развивается как ассистивный продукт: приоритетом остаются безопасный
первый запуск, предсказуемое управление и понятная диагностика на обычном ПК без
консоли.

## Принципы Продукта

- Пользователь не обязан устанавливать Python или пользоваться терминалом.
- Первый запуск должен быть безопасным: без случайных кликов и клавиш.
- Любая проблема должна объясняться в интерфейсе или попадать в support ZIP.
- Linux/Xorg/Wayland различаются явно, без скрытых отказов ввода.
- Новые жесты не должны ухудшать базовый сценарий для людей с ограниченной
  моторикой.
- Релизная сборка проверяется на чистом checkout и на целевых ОС.

## Release Gate

Релиз считается готовым, когда выполнены все пункты:

- CI зелёный.
- `Build AirControl` зелёный на Windows, macOS и Linux.
- Артефакты скачиваются и проходят `tools/verify_release_artifacts.py`.
- Первый запуск открывает стартовое окно без консоли.
- `Safe`/`View` не отправляют input events в ОС.
- Assistive mode работает на машине с камерой.
- Linux tester report содержит понятный вывод по Xorg/Wayland/input backend.
- README, BUILD и USER_GUIDE соответствуют фактическим артефактам.

## Ближайшие Этапы

### 2.3: Качество Первой Настройки

- Встроенный экран `Камера -> Рука -> Ввод -> Готово`.
- GUI-настройка чувствительности, dwell time, dwell radius и cooldown.
- Сохранение нескольких пользовательских профилей.
- Отдельная диагностика “камера видит руку, но ввод ОС заблокирован”.
- Более понятный статус `INPUT RISK` для Wayland и прав macOS Accessibility.

### 2.4: Доступные Способы Управления

- Большая экранная панель действий: левый клик, двойной клик, ПКМ, scroll, pause.
- Расширенный режим минимального движения с мастером подбора активной зоны.
- Scanning menu для пользователей, которым недоступны щипки и быстрые движения.
- Ручная настройка повторов dwell-click для отдельных профилей.
- Export/import профиля пользователя без ручного редактирования `config.json`.

### 2.5: Персонализация

- GUI-мастер записи жеста без терминала.
- Персональный ML-профиль на 5-10 примерах жеста.
- Оценка уверенности жеста в интерфейсе.
- Conflict detector: предупреждение, если два жеста слишком похожи.
- Отдельный мастер установки Vosk model и выбора языка.

### Позже

- Head tracking как fallback, если кисть трудно удерживать в кадре.
- Интеграция с системными accessibility API там, где это даёт реальную пользу.
- Tray/autostart/update-checker без фоновой телеметрии.
- Пресеты для браузера, текстового редактора, презентаций и видеозвонков.
- Подпись Windows/macOS релизов и notarization.

## Что Не Делаем

- Не переписывать ядро на другой язык ради самого переписывания.
- Не добавлять жесты, которые повышают риск случайного клика.
- Не включать облачную телеметрию по умолчанию.
- Не требовать терминал для базовой диагностики.
- Не обещать полную работу Wayland без настройки input backend.
